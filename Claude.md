# Guidelines for Coding Agents

This repository is designed for quick experiments and prototyping. Here are the guidelines for working in this repository:

## Project Structure

- Each experiment should be in its own folder at the root level
- Use descriptive folder names (e.g., `todo-app`, `data-visualization`, `api-experiment`)
- Each web project should have an `index.html` as the entry point

## Code Style

- Prioritize speed and experimentation over perfection
- Keep code simple and readable
- Add comments only where necessary for complex logic
- Don't over-engineer - build what's needed

## Web Projects

For HTML/CSS/JS projects:
- Include an `index.html` file
- Can use inline CSS/JS or separate files as needed
- Projects will be automatically deployed to GitHub Pages

## Python Projects

All Python projects should use `uv` for fast, modern Python package and project management.

### Project Initialization

**For new projects:**
```bash
uv init my-project          # Create a new project
cd my-project
uv add package-name         # Add dependencies
uv run python script.py     # Run scripts
```

**For standalone scripts:**
```bash
uv init --script my-script.py    # Create a standalone script with inline dependencies
uv run my-script.py              # Run with automatic dependency management
```

### Python Version Management

uv automatically manages Python versions - no need to install Python separately:
```bash
uv python install 3.12      # Install specific Python version
uv python pin 3.12          # Pin project to Python 3.12
uv python list              # List available Python versions
```

### Dependency Management

**Add dependencies:**
```bash
uv add requests                    # Add a package
uv add "pytest>=8.0" --dev        # Add dev dependency
uv add ruff --group lint          # Add to a dependency group
```

**Lock and sync:**
```bash
uv lock                     # Update uv.lock file
uv sync                     # Sync environment with lockfile
uv sync --frozen            # Sync without updating lockfile
```

**Run commands:**
```bash
uv run pytest               # Run with project dependencies
uv run --with ruff ruff .   # Run with additional packages
uv run python -m module     # Run as module
```

### Project Structure

Each Python project should have:
- `pyproject.toml` - Project configuration and dependencies (auto-generated by `uv init`)
- `uv.lock` - Locked dependencies (auto-generated, should be committed)
- `.python-version` - Pinned Python version (optional but recommended)
- `README.md` - Setup and usage instructions

### Virtual Environments

uv manages virtual environments automatically, but you can create them manually if needed:
```bash
uv venv                     # Create .venv in current directory
uv venv --python 3.12       # Create with specific Python version
```

### Best Practices

1. **Use `uv run`** instead of activating virtual environments
2. **Commit `uv.lock`** to ensure reproducible installations
3. **Pin Python version** with `uv python pin` for consistency
4. **Use dependency groups** (`--dev`, `--group`) to organize dependencies
5. **Let uv manage Python** - no need to install Python separately
6. **Use `--frozen`** in CI/CD to ensure locked versions are used

### Quick Reference

```bash
uv init [name]              # Create new project
uv add <package>            # Add dependency
uv remove <package>         # Remove dependency
uv sync                     # Install/update dependencies
uv run <command>            # Run command with dependencies
uv python install <ver>     # Install Python version
uv python pin <ver>         # Pin Python version
uv lock                     # Update lockfile
uv tree                     # Show dependency tree
uv build                    # Build package
```

### Advanced Features

- **Scripts with inline dependencies**: Use PEP 723 inline metadata
- **Workspaces**: Manage multiple packages together
- **Platform-specific resolution**: Cross-platform dependency management
- **Tool installation**: `uv tool install ruff` for global tools
- **pip compatibility**: `uv pip install` as pip replacement

## TypeScript Projects

- Include necessary config files (`tsconfig.json`, `package.json`)
- Build outputs should go to a `dist/` folder (already in .gitignore)
- For web projects, ensure compiled output includes `index.html`

## GitHub Pages

- All folders with `index.html` will be accessible via GitHub Pages
- The root `index.html` provides a directory of all projects
- Changes are automatically deployed on push to main branch

## Best Practices

1. Each project is independent - no shared dependencies between projects
2. Document non-obvious setup steps in project-level README files
3. Clean up experiments that are no longer useful
4. Use descriptive commit messages

## Deployment
